name: Sites must be truly personalized and unique
description: Sites look like the same landing page with different images — need unique animations, illustrations, and references per client
type: feedback
---

Sites are coming out as cookie-cutter landing pages that swap images but use identical animations, structures, and references every time.

**Why:** The last 4 sites all used the same 5 reference sites, same marquee testimonials, same glassmorphism cards, same split hero 50/50, same FloatingCTA logic, same word stagger. The user explicitly asked for fairy-themed animations for "A Fada do Dente" and got a generic site instead.

**How to apply:**
- Phase 3 must WebSearch for NEW references relevant to the client's specific concept — not just pick from the static catalog every time
- Phase 6 must create truly unique visual elements tied to the brand (e.g., fairy illustrations, custom SVG animations, themed backgrounds for "A Fada do Dente")
- Each site should have at least 1 "signature element" that could ONLY belong to that client
- Standard patterns (marquee, glassmorphism, split hero) should be varied — not every site needs the same layout
- When the user provides specific creative direction (like "fadas, enchantment, fairy animations"), the pipeline MUST deliver on that
